# Product Requirements Document (PRD): Daily Habit Tracker & AI Coach

## 1. Product Overview
The Daily Habit Tracker is a web application designed to help users build and maintain consistent daily routines. It leverages Generative AI (powered by the Groq API) to analyze user behavior—specifically their completed and missed habits—to provide highly tailored recommendations and encouragement. The app aims to gamify habit building through streaks and progress bars, while utilizing intelligent AI insights to help users overcome roadblocks.

## 2. Target Audience
- Individuals seeking to build new productive routines.
- Users looking to break bad habits by substituting them with positive ones.
- People who benefit from data-driven insights and AI coaching to maintain motivation.

## 3. Core Features

### 3.1 Daily Habit Checklist
- **Functionality**: A clear, accessible list of the user's daily habits.
- **Interaction**: Checkboxes to easily mark tasks as complete or incomplete.
- **Reset**: The list resets daily to allow for tracking on a 24-hour cycle.

### 3.2 "Add Task" Functionality
- **Functionality**: A prominent "Add Task" button allowing users to append new custom habits to their daily list.
- **Flexibility**: Users can define the specific name of the habit (e.g., "Drink 2L Water", "Read 10 pages").

### 3.3 AI-Powered Progress Analysis (via Groq API)
- **Functionality**: An AI assistant that reviews the user's daily/weekly habit performance.
- **Trigger**: The analysis runs based on the ratio of checked vs. unchecked habits.
- **Output**: 
  - **Success**: Personalized encouragement when habits are maintained.
  - **Struggle**: Actionable solutions, adjustments, or root-cause suggestions for frequently unchecked habits.
- **Integration**: Exclusively utilizes the Groq API for fast, reliable LLM generation.

### 3.4 Streak System
- **Functionality**: Tracks consecutive days the user completes *all* their daily tasks.
- **Rewards**: Gamified visual cues or badges to reward the user when streaks are achieved or extended.
- **Motivation**: Designed to prevent "breaking the chain".

### 3.5 Achievement Progress Bar
- **Functionality**: A visual representation of the user's consistency over a given period (e.g., weekly or monthly).
- **Mechanics**: The more consistently tasks are completed day over day, the more the progress bar fills up.
- **Feedback**: Provides immediate gratification and visual proof of long-term effort.

## 4. Technical Requirements
- **Frontend**: A modern web framework (e.g., React, Next.js, Vite/Vue, or plain HTML/TailwindCSS) focusing on responsive design and smooth micro-interactions.
- **Backend**: Python (e.g., FastAPI, Flask, or Django) to securely handle the integration with the Groq API and manage user data.
- **AI Model Provider**: Groq API (e.g., Llama 3 or Mixtral through Groq for high-speed inference).
- **Database**: Local storage or a lightweight database (e.g., SQLite, PostgreSQL) to store habits, completion history, streaks, and AI analysis logs.

## 5. User Journey
1. **Onboarding**: User lands on the app and adds their first few daily habits via the "Add Task" button.
2. **Daily Use**: User visits the web app daily to check off completed habits.
3. **Tracking**: Upon checking items, the progress bar updates, and the current streak count evaluates.
4. **Insights**: At the end of the day/week, or dynamically on the dashboard, the AI coach presents an analysis of what is working well and advice on how to tackle neglected habits.

## 6. Future Considerations (V2)
- Authentication and cross-device syncing.
- Habit categories (Health, Work, Personal).
- Weekly and monthly detailed PDF/Email reports generated by AI.
